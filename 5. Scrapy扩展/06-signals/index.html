Signals(信号)

Scrapy广泛地使用信号来通知某些事件的发生。您可以在Scrapy项目中捕捉到一些信号(例如使用extension(扩展))，以执行附加任务或扩展Scrapy以添加不是开箱即用的功能。

即使信号提供了几个参数，捕获它们的处理程序也不需要接受所有参数 - 信号调度机制只会交付处理程序接收的参数。

您可以通过Signals API连接到信号(或发送自己的)。

下面是一个简单的例子，展示了如何捕捉信号并执行一些操作:

from scrapy import signals
from scrapy import Spider


class DmozSpider(Spider):
    name = "dmoz"
    allowed_domains = ["dmoz.org"]
    start_urls = [
        "http://www.dmoz.org/Computers/Programming/Languages/Python/Books/",
        "http://www.dmoz.org/Computers/Programming/Languages/Python/Resources/",
    ]


    @classmethod
    def from_crawler(cls, crawler, *args, **kwargs):
        spider = super(DmozSpider, cls).from_crawler(crawler, *args, **kwargs)
        crawler.signals.connect(spider.spider_closed, signal=signals.spider_closed)
        return spider


    def spider_closed(self, spider):
        spider.logger.info('Spider closed: %s', spider.name)


    def parse(self, response):
        pass

延迟的信号处理程序

有些信号支持从处理程序返回Twisted deferreds，请参阅下面的内置信号参考以了解哪些信号。

内置信号参考

下面是一些Scrapy的内置信号及其含义。

engine_started

scrapy.signals.engine_started()

当Scrapy engine开始抓取时发送。

此信号支持从处理程序返回延迟。

这个信号可能在'spider_opened'信号之后触发，这取决于spider是如何启动的。
所以不要依赖这个信号在'spider_open'之前被触发。

engine_stopped

scrapy.signals.engine_stopped()

当Scrapy engine停止时发送(例如，抓取过程完成时)。

此信号支持从处理程序返回延迟。

item_scraped

scrapy.signals.item_scraped(item, response, spider)

当一个item被抓取后，在它通过所有的Item Pipeline阶段(没有被丢弃)后发送。

此信号支持从处理程序返回延迟。

参数:

  • item (字典或Item 对象) – 被抓取的item
  • spider (Spider 对象) – 抓取item的spider
  • response (Response 对象) – 从item被抓取的地方得到的response

item_dropped

scrapy.signals.item_dropped(item, response, exception, spider)

当某个阶段引发DropItem异常时，item从Item Pipeline中删除后发送。

此信号支持从处理程序返回延迟。

参数:

  • item (字典或Item 对象) – 从Item Pipeline中删除的item
  • spider (Spider 对象) – 抓取item的spider
  • response (Response 对象) – 从item被丢弃的地方得到的response
  • exception (DropItem 异常) – item被丢弃导致的异常((必须是DropItem子类))

spider_closed

scrapy.signals.spider_closed(spider, reason)

在spider被关闭后发出的。这可以用来释放在spider_opened上保留的每个spider资源

此信号支持从处理程序返回延迟。

参数:

  • spider (Spider 对象) – 已被关闭的spider
  • reason (字符串) – 一种描述spider被关闭的原因的字符串。如果它被关闭了，因为蜘蛛已经完成了抓取，reason是'finished'了。否则，如果spider是通过调用close_spider engine方法手动关闭的，那么reason就在于该方法的reason（默认为'cancelled'）。如果engine被关闭（例如，按Ctrl-C停止它），原因将是'shutdown'。

spider_opened

scrapy.signals.spider_opened(spider)

在spider被打开且开始抓取的时候发送, 这通常用于保存每个spider的资源，但是可以用于任何需要在spider打开时执行的任务。

此信号支持从处理程序返回延迟。

参数:

  • spider (Spider 对象) – 已经被打开的spider

spider_idle(空闲)

scrapy.signals.spider_idle(spider)

当spider空闲时发送，这意味着spider没有进一步发展：

  • 等待下载的requests
  • requests安排
  • 在item pipeline中处理的items

如果这个signal的所有处理程序完成后，空闲状态仍然存在，那么引擎就会开始关闭spider。在spider完成关闭后，将发出spider_closed信号。

您可以raise一个DontCloseSpider异常，以防止spider被关闭。

这个信号不支持从操作者返回的延迟。

参数:

  • spider (Spider 对象) – 空闲的spider
在您的'[spider_idle](#)'处理程序中安排一些requests并'不能'保证它可以防止spider被关闭，
尽管有时可以。这是因为如果所有的预定requests都被scheduler拒绝了（例如，由于重复而被过滤），
那么spider可能仍然处于空闲状态。

spider_error

scrapy.signals.spider_error(failure, response, spider)

当spider callback产生错误时（即。rasie了一个异常)。

这个信号不支持从操作者返回的延迟。

参数:

  • failure (Failure 对象) – 作为一个Twisted Failure对象所引发的异常

  • response (Response 对象) – 当异常被抛出时正在处理的response

  • spider (Spider object) – 引发异常的spider

request_scheduled

scrapy.signals.request_scheduled(request, spider)

当engine schedule一个Request时发送，稍后再下载。

这个信号不支持从他们的处理程序返回的延迟。

参数:

  • request (Request 对象) – 到达scheduler的request
  • spider (Spider object) – yield出去的request

request_dropped

scrapy.signals.request_dropped(request, spider)

当由engine调度的Request稍后被下载时，被scheduler拒绝时发送。

这个信号不支持从他们的处理程序返回的延迟。

参数:

  • request (Request 对象) – 到达scheduler的request
  • spider (Spider 对象) – yield出去的request

response_received

scrapy.signals.response_received(response, request, spider)

当engine从downloader接收到一个新的Response时发出。

这个信号不支持从操作者返回的延迟。

参数:

  • response (Response 对象) – 接收到的response
  • request (Request 对象) – 产生response的request
  • spider (Spider 对象) – 用于response的spider

response_downloaded

scrapy.signals.response_downloaded(response, request, spider)

下载HTTPResponse后立即由downloader发送。

此信号不支持从其处理程序返回延迟。

参数:

  • response (Response 对象) – 下载的resposne
  • request (Request 对象) – 生成response的request
  • spider (Spider 对象) – 用于response的spider
