Link Extractors(链接提取器)

Link extractors是只用于从web页面(scrapy.http.Response对象)中提取链接的对象，最终将被跟踪。

scrapy.linkextractors.LinkExtractor可以在Scrapy中使用，但是您可以通过一个简单的接口创建自己的自定义Link Extractors来满足您的需求。

每个link extractor都拥有的惟一公共方法是extract_links，它接收一个Response对象并返回一个scrapy.link.Link对象列表。Link extractors只需要实例化一次，它们的extract_links方法要调用好几次，使用不同的responses提取要跟踪的链接。

Link extractors用于CrawlSpider类(Scrapy中可用)，通过一组规则，但是您也可以在spiders中使用它，即使您没有从CrawlSpider子类化它，因为它的目的非常简单:提取链接。

内置link extractors参考

scrapy.linkextractors模块中提供了与Scrapy绑定的Link extractors类。

默认的link extractor是LinkExtractor，与LxmlLinkExtractor相同:

from scrapy.linkextractors import LinkExtractor

在以前的Scrapy按本中曾经有其他的link extractor类，但是现在它们已经被废弃了。

LxmlLinkExtractor

scrapy.linkextractors.lxmlhtml.LxmlLinkExtractor(allow=(), deny=(), allow_domains=(), deny_domains=(), deny_extensions=None, restrict_xpaths=(), restrict_css=(), tags=('a', 'area'), attrs=('href', ), canonicalize=False, unique=True, process_value=None, strip=True) 这是一个类(class)

LxmlLinkExtractor是推荐的具有方便过滤选项的link extractor。它是使用健壮的lxml HTMLParser实现的。

参数:

  • allow (正则表达式(或列表)) – 单个正则表达式(或正则表达式列表), (绝对)url必须匹配才能被提取, 如果没有给出(或空)，它将匹配所有链接。
  • deny (正则表达式(或列表)) - 单个正则表达式(或正则表达式列表), (绝对)url必须匹配才能被排除。(既,不提取)。 它优先于allow参数, 如果没有给出(或空)，它不会排除任何链接。
  • allow_domains (字符串或列表) - 用于提取链接的单个值或包含domains的字符串列表
  • deny_domains (字符串或列表) - 包含domains的单个值或字符串列表，这些domains不用于提取链接
  • deny_extensions (列表) - 包含在提取链接时应忽略的extensions(扩展)的单个值或字符串列表。如果没有指定，它将默认设置为scrapy中定义的IGNORED_EXTENSIONS列表在scrapy.linkextractors包下。
  • restrict_xpaths (字符串或列表) – 一个XPath(或XPath的列表)，它定义了response中的链接应该从哪里提取。 如果给定，则只扫描这些XPath选择的文本以查找链接。请参见下面的例子。
  • restrict_css (字符串或列表) – 一个CSS选择器(或selectors列表)，它定义了response中的链接应该从哪里提取。 具有与restrict_xpaths相同的行为。
  • tags (字符串或列表) – 提取链接时要考虑的标记或标记列表。 默认为('a'， 'area')。
  • attrs (列表) – 在寻找要提取的链接时应该考虑的属性或属性列表(仅适用于tags参数中指定的标记), 默认为('href')
  • canonicalize (布尔类型) – 规范化每个提取的url(使用w3lib.url.canonicalize_url), 默认为False, 注意，canonicalize_url是用于重复检查的: 它可以改变在服务器端可见的URL, 因此，对于具有规范化和原始url的requests，response可能不同, 如果您使用LinkExtractor来跟踪链接，那么保持默认设置canonicalize=False会更加健壮
  • unique (布尔类型) – 是否对提取的链接应用重复过滤。
  • process_value (可调用的) – 一个函数，它接收从已扫描的tag和attributes中提取的每个值，并且可以修改这个值并返回一个新的值，或者返回None以完全忽略链接。 如果没有给出，process_value默认为lambda x: x

例如，从这段代码中提取链接:

<a href="javascript:goToPage('../other/page.html'); return false">Link text</a>

您可以在process_value中使用以下函数:

def process_value(value): m = re.search("javascript:goToPage\('(.*?)'", value) if m: return m.group(1)

  • strip (布尔类型) – 是否从提取的attributes中删除空白, 根据HTML5标准，必须从<a>、<area>等元素的href属性、<img>、<iframe>元素的src属性中去掉开头和结尾的空白, 等等，所以默认情况下，LinkExtractor会在默认情况下删除空格字符, 设置strip=False来关闭它(例如，如果你正在从elements或attributes中提取url，这些elements或attributes允许开头/结尾有空格)
