Feed exports(导出各种格式文件)

最新版本0.10

实现抓取器最经常需要的特性之一是能够正确地存储被抓取的数据，这通常意味着使用被抓取的数据(通常称为“export feed”)生成一个“导出文件”，以供其他系统使用。

Scrapy通过Feed Exports提供了这种开箱即用的功能，它允许您抓取item生成feed并使用多种序列化格式或者进行存储后端。

序列化

对于序列化抓取的数据，feed exports使用Item exporters。这些格式支持开箱即用:

  • JSON
  • JSON lines
  • CSV
  • XML

但是您也可以通过FEED_EXPORTERS设置来扩展支持的格式。

JSON

  • FEED_FORMAT: json
  • Exporter used: JsonItemExporter
  • 如果您使用的是带有大量feeds的JSON，请参阅此警告。

JSON lines

  • FEED_FORMAT: jsonlines
  • Exporter used: JsonLinesItemExporter

CSV

  • FEED_FORMAT: csv
  • Exporter used: JsonItemExporter
  • 要指定要导出的列及其顺序，请使用FEED_EXPORT_FIELDS。其他feed exporters也可以使用此选项，但它对于CSV很重要，因为与许多其他导出格式不同，CSV使用固定的头。

XML

  • FEED_FORMAT: xml
  • Exporter used: XmlItemExporter

Pickle

  • FEED_FORMAT: pickle
  • Exporter used: PickleItemExporter

Marshal

  • FEED_FORMAT: marshal
  • Exporter used: MarshalItemExporter

存储设备

在使用feed exports时，使用URI(通过FEED_URI设置)定义存储feed的位置。feed exports支持由URI scheme定义的多个存储后端类型。

支持开箱即用的存储后端有:

  • Local filesystem
  • FTP
  • S3(需要botocore或boto)
  • Standard output

如果所需的外部库不可用时，一些存储后端可能不可用。例如，S3后端仅在安装了botocore或boto库时可用(Scrapy仅在Python 2上支持boto)。

存储URI参数

存储URI还可以包含在创建feed时替换feed的参数。这些参数是:

  • %(time)s - 在创建feed时被时间戳替换
  • %(name)s - 被替换为spider名称

任何其他命名参数都会被相同名称的spider属性替换。例如，创建feed的那一刻%(site_id)将被替换为spider.site_id属性

这里有一些例子来说明:

  • 存储在FTP上, 每个spider使用一个目录
    • ftp://user:password@ftp.example.com/scraping/feeds/%(name)s/%(time)s.json
  • 存储在S3上, 每个spider使用一个目录
    • s3://mybucket/scraping/feeds/%(name)s/%(time)s.json

存储后端

本地文件系统

存储URI还可以包含参数

  • URI scheme: file
  • URI例子: file:///tmp/export.csv
  • 需要外部库: none

注意，对于本地文件系统存储(仅)，如果指定了/tmp/export.csv这样的绝对路径，则可以省略该方案。但这只适用于Unix系统。

FTP

存储在FTP服务器上的feeds

  • URI scheme: ftp
  • URI例子: ftp://user:pass@ftp.example.com/path/to/export.csv
  • 需要外部库: none

S3

存储在亚马逊S3上的feeds

  • URI scheme: s3

  • URI例子:

    • s3://mybucket/path/to/export.csv
    • s3://aws_key:aws_secret@mybucket/path/to/export.csv

  • 需要外部库: botocore (Python 2 and Python 3)或boto (Python 2 only)

AWS凭证可以在URI中作为用户/密码传递，也可以通过以下设置传递:

  • AWS_ACCESS_KEY_ID
  • AWS_SECRET_ACCESS_KEY

标准输出

feed被写入到Scrapy处理的标准输出。

  • URI scheme: stdout
  • URI例子: stdout:
  • 需要外部库: none

设置

这些设置用于配置feed exports:

  • FEED_URI (强制)
  • FEED_FORMAT
  • FEED_STORAGES
  • FEED_EXPORTERS
  • FEED_STORE_EMPTY
  • FEED_EXPORT_ENCODING
  • FEED_EXPORT_FIELDS
  • FEED_EXPORT_INDENT

FEED_URI

默认: None

导出feed的URI。有关受支持的URI方案，请参阅存储后端。

启用feed_exports需要此设置。

FEED_FORMAT

用于feed格式的序列化。有关可能的值，请参阅序列化格式。

FEED_EXPORT_ENCODING

默认: None

feed的编码。

如果未设置或设置为None(默认)，除了JSON输出，它对所有内容都使用UTF-8，出于历史原因，JSON输出使用安全的数字编码(\uXXXX序列)。

如果要为JSON使用utf-8，也可以使用utf-8。

FEED_EXPORT_FIELDS

默认: None

要导出的字段列表，可选。例子:

FEED_EXPORT_FIELDS = ["foo", "bar", "baz"]

使用FEED_EXPORT_FIELDS选项定义要导出的字段及其顺序。

当FEED_EXPORT_FIELDS为空或为None(默认)时，Scrapy使用dicts中定义的字段或spider生成的Item子类。

如果导出者需要一组固定的字段(这是CSV导出格式的情况)，且FEED_EXPORT_FIELDS为空或为None，那么Scrapy尝试从导出的数据推断字段名 —— 目前它使用的字段名来自第一个item。

FEED_EXPORT_INDENT

默认: 0

用于缩进每个级别上的输出的空间量。如果feed_export_缩进是一个非负整数，那么数组元素和对象成员将用缩进级别漂亮地打印出来。缩进级别为0(默认值)或负值，将把每个项放到新行中。None选择最紧凑的表示形式。

目前只由jsonitemexports和xmlitemexports实现，即当您导出到.json或.xml时。

FEED_STORE_EMPTY

默认: False

是否导出空的feed(即。没有item的feed)。

FEED_STORAGES

默认: {}

包含您的项目支持的额外feed存储后端的dict。 键是URI schemes，值是存储类的路径。

FEED_STORAGES_BASE

{
    '': 'scrapy.extensions.feedexport.FileFeedStorage',
    'file': 'scrapy.extensions.feedexport.FileFeedStorage',
    'stdout': 'scrapy.extensions.feedexport.StdoutFeedStorage',
    's3': 'scrapy.extensions.feedexport.S3FeedStorage',
    'ftp': 'scrapy.extensions.feedexport.FTPFeedStorage',
}

一个字典，包含由Scrapy支持的内置feed存储后端。您可以通过在FEED_STORAGES中为它们的URI scheme分配None来禁用任何这些后端。例如，要禁用内置FTP存储后端(不需要替换)，请将其放在您的settings.py中。

FEED_STORAGES = {
    'ftp': None,
}

FEED_EXPORTERS

默认: {}

包含由您的项目支持的额外exporters的一种dict。键是序列化格式，值是Item exporter类的路径。

FEED_EXPORTERS_BASE

默认:

{
    'json': 'scrapy.exporters.JsonItemExporter',
    'jsonlines': 'scrapy.exporters.JsonLinesItemExporter',
    'jl': 'scrapy.exporters.JsonLinesItemExporter',
    'csv': 'scrapy.exporters.CsvItemExporter',
    'xml': 'scrapy.exporters.XmlItemExporter',
    'marshal': 'scrapy.exporters.MarshalItemExporter',
    'pickle': 'scrapy.exporters.PickleItemExporter',
}

包含由Scrapy支持的内置feed exporters的一种dict。您可以通过在FEED_EXPORTERS中为它们的序列化格式分配None来禁用这些exporters中的任何一个。例如，要禁用内置的CSV导出器(不需要替换)，请将其放在您的settings.py:

FEED_EXPORTERS = {
    'csv': None,
}
