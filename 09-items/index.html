Items

抓取的主要目标是从非结构化数据源(通常是web页面)中提取结构化数据。Scrapy spider可以以Python字典的形式返回提取的数据。虽然Python字典很方便也很熟悉，但它缺乏结构:在字段名中很容易出现拼写错误或返回不一致的数据，特别是在具有许多spiders的大型项目中。

要定义通用的输出数据格式，Scrapy提供了Item类。Item对象是用于收集Scrapy抓取数据的简单容器。它们提供了一个类似于字典的API，具有方便的语法来声明它们的可用字段。

各种各样的Scrapy组件使用items提供额外的信息, exporters查看声明的字段以确定要导出的列，可以使用item字段metadata定制序列化，trackref跟踪item实例以帮助查找内存泄漏(请参阅使用trackref调试内存泄漏)，等等。

定义Items

这是一个例子, 使用简单的类定义语法和Field对象定义Items:

import scrapy
class Product(scrapy.Item):
    name = scrapy.Field()
    price = scrapy.Field()
    stock = scrapy.Field()
    last_updated = scrapy.Field(serializer=str)

注意! 熟悉Django的人会注意到，Scrapy Items与Django模型类似，但由于没有不同字段类型的概念，Scrapy Items要简单得多。

Item Fields

Field对象用于为每个字段指定metadata。例如，上面示例中演示了last_updates字段的序列化。

您可以为每个字段指定任何类型的metadata。Field对象接受的值没有限制。出于同样的原因，没有所有可用的metadata键的引用列表。Field对象中定义的每个键都可以由不同的组件使用，并且只有那些组件知道它。您还可以根据自己的需要在项目中定义和使用任何其他Field键。Field对象的主要目标是提供一种方法来在一个位置定义所有字段的metadata 。通常，那些行为依赖于每个字段的组件使用某些字段键来配置该行为。您必须参考它们的文档，以查看每个组件使用哪些metadata键。

需要注意的是，用于定义Field对象不保留为类属性。相反，可以通过Item.fields访问它们属性。

处理Items

下面是使用上面定义的Productitem执行的常见任务的一些示例。您会注意到这个API与pyton的字典API非常相似。

创建items

>>> product = Product(name='Desktop PC', price=1000)
>>> print product
Product(name='Desktop PC', price=1000)

获取field(字段)的值

>>> product['name']
Desktop PC
>>> product.get('name')
Desktop PC

>>> product['price']
1000

>>> product['last_updated']
Traceback (most recent call last):
    ...
KeyError: 'last_updated'

>>> product.get('last_updated', 'not set')
not set

>>> product['lala'] # 获取位置的字段
Traceback (most recent call last):
    ...
KeyError: 'lala'

>>> product.get('lala', 'unknown field')
'unknown field'

>>> 'name' in product  # name在这个类中是否被填充过?
True

>>> 'last_updated' in product  # last_updated在这个类中是否被填充过?
False

>>> 'last_updated' in product.fields  # last_updated是否是一个定义过的字段?
True

>>> 'lala' in product.fields  # lala是否是一个定义过的字段?
False

设置field(字段)值

>>> product['last_updated'] = 'today'
>>> product['last_updated']
today

>>> product['lala'] = 'test' # 设置未知的字段
Traceback (most recent call last):
    ...
KeyError: 'Product does not support field: lala'

访问所有填充的值

要访问所有填充的值，只需使用典型的python字典API:

>>> product.keys()
['price', 'name']

>>> product.items()
[('price', 1000), ('name', 'Desktop PC')]

其他常见的任务

从items中创建字典

>>> dict(product) # 用填充的值创建一个字典
{'price': 1000, 'name': 'Desktop PC'}

扩展Items

可以通过定义原始Item的子类来扩展Items(添加更多字段或更改某些字段的metadata)。

例如:

class DiscountedProduct(Product):
    discount_percent = scrapy.Field(serializer=str)
    discount_expiration_date = scrapy.Field()

您还可以通过使用以前的metadata字段来扩展字段metadata，并追加更多的值，或者更改现有的值，如下所示:

class SpecificProduct(Product):
    name = scrapy.Field(Product.fields['name'], serializer=my_serializer)

这将为name字段添加(或替换)serializer的metadata键，保留所有以前存在的metadata值。

Item对象

scrapy.item.Item([arg]) 这是一个类(class)

通过给定参数返回初始化的新Item。

Items复制标准的python字典API，包括其构造函数。Items提供的唯一附加属性是:

fields

包含此Item所有定义过fields的字典，而不仅仅是已填充的字段。键是字段名，值是Item定义中使用的Field对象。

Field对象

scrapy.item.Field([arg]) 这是一个类(class)

Field类只是内置python dict 类的别名，不提供任何额外的功能或属性。换句话说，Field对象普遍是python的字典。用于支持基于类属性的item定义语法的单独的一个类。
