Debugging Spiders(调试Spiders)

本文介绍调试spider的最常见技术。考虑以下的scrapy spider:

import scrapy
from myproject.items import MyItem

class MySpider(scrapy.Spider):
    name = 'myspider'
    start_urls = (
        'http://example.com/page1',
        'http://example.com/page2',
        )

    def parse(self, response):
        # 收集 `item_urls`
        for item_url in item_urls:
            yield scrapy.Request(item_url, self.parse_item)

    def parse_item(self, response):
        item = MyItem()
        # 填充 `item` 字段
        # 并且提取item_details_url
        yield scrapy.Request(item_details_url, self.parse_details, meta={'item': item})

    def parse_details(self, response):
        item = response.meta['item']
        # 填充更多的 `item` 字段
        return item

基本上，这是一个简单的spider，它解析两页items(start_urls)。item还有一个包含附加信息的详细信息页面，因此我们使用Request的meta功能传递部分填充的item。

Parse命令

检查spider输出的最基本方法是使用parse命令。它允许在方法级别检查spider的不同部分的行为。它的优点是灵活且易于使用，但不允许在方法内部调试代码。

为了看到从一个特定的url抓取的项目:

$ scrapy parse --spider=myspider -c parse_item -d 2 <item_url>
[ ... scrapy log lines crawling example.com spider ... ]

>>> STATUS DEPTH LEVEL 2 <<<
# Scraped Items  ------------------------------------------------------------
[{'url': <item_url>}]

# Requests  -----------------------------------------------------------------
[]

使用-verbose或-v选项，我们可以看到在每个深度级别的状态:

$ scrapy parse --spider=myspider -c parse_item -d 2 -v <item_url>
[ ... scrapy log lines crawling example.com spider ... ]

>>> DEPTH LEVEL: 1 <<<
# Scraped Items  ------------------------------------------------------------
[]

# Requests  -----------------------------------------------------------------
[<GET item_details_url>]


>>> DEPTH LEVEL: 2 <<<
# Scraped Items  ------------------------------------------------------------
[{'url': <item_url>}]

# Requests  -----------------------------------------------------------------
[]

检查从单个start_url抓取下来的items，也可以很容易地实现使用:

$ scrapy parse --spider=myspider -d 3 'http://example.com/page1'

Scrapy Shell

虽然parse命令对于检查spider的行为非常有用，但是除了显示接收到的响应和输出之外，检查callback内部发生了什么并没有什么帮助。当parse_details有时不接收item时，如何调试这种情况?

幸运的是，在这种情况下，shell就是您的面包和黄油(参见调用spider中的shell来检查response):

from scrapy.shell import inspect_response

def parse_details(self, response):
    item = response.meta.get('item', None)
    if item:
        # 填充更多 `item` 字段
        return item
    else:
        inspect_response(response, self)

参见:从spider调用shell以检查response。

打开浏览器

有时候，您只是想看看浏览器中的某个resposne是什么样子的，您可以使用open_in_browser函数进行处理。下面是一个如何使用它的例子:

from scrapy.utils.response import open_in_browser

def parse_details(self, response):
    if "item name" not in response.body:
        open_in_browser(response)

open_in_browser将打开一个浏览器，其中包含从Scrapy接收到的response，并调整base tag，以便正确显示图像和样式。

Logging

Logging是获取有关spider运行信息的另一个有用选项。虽然不那么方便，但它的优点是，如果需要日志，在以后的所有运行中都可以使用:

def parse_details(self, response):
    item = response.meta.get('item', None)
    if item:
        # 填充更多 `item` 字段
        return item
    else:
        self.logger.warning('No item received for %s', response.url)

有关更多信息，请参阅Logging部分。
