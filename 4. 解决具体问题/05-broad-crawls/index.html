Broad Crawls

Scrapy默认是针对抓取特定站点进行优化的。这些站点通常由一个Scrapy spider处理，尽管这不是必须的或必需的(例如，有一些通用的spider，可以处理向它们抛出的任何给定站点)。

除了这种“聚焦爬虫”,还有另一个常见的爬行涵盖数量大(可能是无限)的domain,并且仅受时间或其他任意约束的限制，而不是在domain爬行到完成时停止或者当没有更多的请求来执行。这些被称为“广泛爬行”，是搜索引擎使用的典型的爬虫。

这些是一些常见的性质，经常在广泛的爬行发现:

  • 它们搜索许多domain(通常,无限)，而不是特定的一组站点
  • 它们并不一定要遍历domain才能完成，因为这样做是不实际的(或不可能的)，而是应该根据抓取的时间或页数来限制抓取
  • 它们在逻辑上更简单(与具有许多提取规则的非常复杂的爬行器相反)，因为数据通常在单独的阶段进行后期处理
  • 它们可以同时爬行多个domain，这使得它们不受任何特定站点约束的限制，从而获得更快的爬行速度(为了礼貌起见，每个站点都是缓慢爬行的，但是许多站点是并行爬行的)

如上所述，Scrapy默认设置是针对有重点的spider而不是广泛的爬虫进行优化的。但是，由于其异步体系结构，Scrapy非常适合执行快速广泛的爬行。这个页面总结了在使用Scrapy进行广泛爬行时需要记住的一些内容，并提供了一些Scrapy settings关于如何进行优化以实现高效的广泛爬行的具体建议。

提高并发性

并发性是并行处理的请求数。有一个全局限制和一个针对每个domain限制。

Scrapy中的默认全局并发限制不适用于并行地爬行许多不同的domain，因此您需要增加它。增加多少取决于爬虫程序有多少CPU可用。一个好的起点是100，但最好的方法是做一些试验，并确定什么并发性会限制你的Scrapy处理过程。为了获得最佳性能，您应该选择CPU使用率在80-90%的并发性。

增加全局并发性的使用:

CONCURRENT_REQUESTS = 100

增加Twisted IO线程池的最大容量

目前，Scrapy使用线程池以阻塞的方式进行DNS解析。随着并发级别的提高，爬行速度可能会变慢，甚至失败，导致DNS解析超时。可能的解决方案是增加处理DNS查询的线程数。DNS队列将被更快地处理，加快连接的建立和爬行。

增加最大线程池大小使用:

REACTOR_THREADPOOL_MAXSIZE = 20

设置您自己的DNS

如果您有多个爬虫进程和一个中央DNS，它可能会像DoS攻击一样攻击DNS服务器，导致整个网络速度减慢，甚至阻塞您的计算机。为了避免这种情况，您可以使用本地缓存和上游到一些大型DNS服务器，如OpenDNS或Verizon。

减少日志级别

在进行广泛的爬行时，您通常只对所获得的爬行速度和发现的任何错误感兴趣。这些数据是由Scrapy在使用INFO日志级别时报告的。为了节省CPU(和日志存储需求)，在生产环境中预生成大范围爬虫时，不应该使用DEBUG日志级别。不过，在开发(广泛的)爬虫程序时使用DEBUG级别可能不错。

设置log级别使用:

LOG_LEVEL = 'INFO'

禁用Cookies

除非您确实需要，否则禁用cookie。在进行广泛的爬虫时，通常不需要cookie(搜索引擎爬虫会忽略它们)，它们通过节省CPU周期和减少scrapy爬虫的内存占用来提高性能。

禁用cookies使用:

COOKIES_ENABLED = False

禁用重试

重新失败的HTTP请求会大大降低爬行速度，特别是当站点原因响应非常慢(或失败)时，会导致超时错误，导致多次不必要地重试，从而阻止爬行器容量被其他domain重用。

禁用重试:

RETRY_ENABLED = False

减少下载超时

除非您正在从一个非常慢的连接中爬行(对于广泛的爬行来说不应该是这样的)，否则应该将减少下载超时，以便快速丢弃被卡住的请求，并释放出处理下一个请求的能力。

为了减少下载超时使用

DOWNLOAD_TIMEOUT = 15

禁用重定向

考虑禁用重定向，除非您有兴趣跟踪它们。在进行广泛的爬行时，通常在稍后的爬行中重新访问站点时保存重定向并解决它们。这也有助于保持每次抓取批处理的请求数量不变，否则重定向循环可能会导致爬虫程序在任何特定domain上占用太多的资源。

禁止重定向使用

REDIRECT_ENABLED = False

启用“Ajax Crawlable页面”的爬行

一些页面(根据2013年的经验数据，最高达到1%)声明自己是ajax crawlable。这意味着它们提供的内容的纯HTML版本通常只能通过AJAX获得。页面可以用两种方式表示:

  1. 在URL使用#! - 这是默认的方式
  2. 使用特殊的meta tag - 这种方法在“main”、“index”网站页面上使用。

Scrapy handles

  1. 自动处理
  2. 激活AjaxCrawlMiddleware
AJAXCRAWL_ENABLED = True

当进行广泛的爬行时，通常会抓取大量的“index”web页面;

AjaxCrawlMiddleware有助于正确地抓取它们。默认情况下，它是关闭的，因为它有一些性能开销，启用它来聚焦爬虫没有多大意义。
