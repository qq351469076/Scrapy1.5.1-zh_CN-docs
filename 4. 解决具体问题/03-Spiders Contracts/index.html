Spiders Contracts

最新版本0.15

这是一个新特性(在Scrapy 0.15中介绍)，可能会受到小功能/API更新的影响。
检查[发布记录](https://docs.scrapy.org/en/latest/news.html#news)，以通知更新。

测试spider可能会特别烦人，虽然没有什么可以阻止您编写单元测试，但任务会很快变得很麻烦。Scrapy提供了一种通过contract的方式来测试您的spider的综合方法。

这允许您通过硬编码示例url来测试spider的每个callback，并检查callback如何处理response的各种约束。每个contract的前缀都是@，并包含在文档字符串中。请看下面的例子:

def parse(self, response):
    """ 该函数解析了一个简单的response. 有些contract与这个文档字符串混杂在一起.

    @url http://www.amazon.com/s?field-keywords=selfish+gene
    @returns items 1 16
    @returns requests 0 0
    @scrapes Title Author Year Price
    """

此callback使用三个内置contracts进行测试:

scrapy.contracts.default.UrlContract 这是一个类(class)

此contract(@url)设置在检查此spider的其他contract条件时使用的示例url。这份contract是强制性的。在运行检查时，所有缺少contract的callback都被忽略:

@url url

scrapy.contracts.default.ReturnsContract 这是一个类(class)

这个contract(@returns)为spider返回的items和requests设置了下限和上限。上限是可选的:

@returns item(s)|request(s) [min [max]]

scrapy.contracts.default.ScrapesContract 这是一个类(class)

这个contract(@scrapes)检查callback返回的所有items是否具有指定的字段:

@scrapes field_1 field_2 ...

自定义Contracts

如果您发现您需要比内置的scrapy contracts更强大的功能，您可以使用SPIDER_CONTRACTS settings在项目中创建和加载您自己的contracts:

SPIDER_CONTRACTS = {
    'myproject.contracts.ResponseCheck': 10,
    'myproject.contracts.ItemValidate': 10,
}

每个合同必须从scrapy.contracts.Contract继承，并可以覆盖三个方法:

scrapy.contracts.Contract(method, *args) 这是一个类(class)

参数:

  • method (方法) – 与contract相关联的callback函数
  • args (列表) – 传入docstring的参数列表(空格分隔)

adjust_request_args(args)

这将接收一个dict作为一个参数，其中包含Request对象的默认参数。必须返回它的相同或修改版本。

pre_process(response)

这允许在将response传递给callback之前，对从实例requests接收到的response进行各种检查。

post_process(output)

这允许处理callback的输出。迭代器在传递到此钩子之前被转换成listified。

下面是一个contract demo，它检查接收到的response中是否存在自定义标头。抛出scrapy.exceptions.ContractFail是为了让失败被很好地打印出来::

from scrapy.contracts import Contract
from scrapy.exceptions import ContractFail

class HasHeaderContract(Contract):
    """ 演示contract, 检查自定义的标头是否存在
        @has_header X-CustomHeader
    """

    name = 'has_header'

    def pre_process(self, response):
        for header in self.args:
            if header not in response.headers:
                raise ContractFail('X-CustomHeader not present')
