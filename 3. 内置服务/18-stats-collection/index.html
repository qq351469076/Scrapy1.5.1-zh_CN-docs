Stats Collection(状态收集)

Scrapy提供了一个以键/值的形式收集状态的方便工具，其中值通常是计数器。该工具被称为Stats Collector，可以通过Crawler API的stats属性访问它，下面一节中常见的Stats Collector使用的例子就说明了这一点。

但是，Stats Collector始终是可用的，因此您总是可以在模块中导入它并使用它的API(用于递增或设置新的stat键)，而不管stats collection是否启用。如果它被禁用，API仍然可以工作，但是它不会收集任何东西。这是为了简化stats collector的使用:在spider、Scrapy extension或使用Stats Collector的任何代码中，您应该只使用一行代码来收集stats。

Stats Collector的另一个特性是，启用时非常高效(在启用时)，禁用时非常高效(几乎不需要注意)。

Stats Collector为每个打开的spider保留一个stats表，在spider打开时自动打开，在spider关闭时关闭。

使用常用的Stats Collector

通过stats属性访问stats collector。下面是一个访问统计数据的扩展示例:

class ExtensionThatAccessStats(object):

    def __init__(self, stats):
        self.stats = stats

    @classmethod
    def from_crawler(cls, crawler):
        return cls(crawler.stats)

设置stat的值

stats.set_value('hostname', socket.gethostname())

增加stat的值

stats.inc_value('custom_count')

只有在大于之前的值时才设置stat值:

stats.max_value('max_items_scraped', value)

只有在低于之前的值时才设置stat值:

stats.min_value('min_free_memory_percent', value)

获得stat的值

>>> stats.get_value('custom_count')
1

得到所有stat

>>> stats.get_stats()
{'custom_count': 1, 'start_time': datetime.datetime(2009, 7, 14, 21, 47, 28, 977139)}

可用的Stats Collectors

除了基本的StatsCollector，在Scrapy中还有其他的StaStats Collectors可以扩展基本的Stats Collector。您可以通过STATS_CLASS setting选择使用哪个Stats Collector。默认的Stats Collector是MemoryStatsCollector。

MemoryStatsCollector

scrapy.statscollectors.MemoryStatsCollector 这是一个类(class)

一个简单的stats collector，它在每次抓取(对于每个spider)结束后，将最后一次抓取运行的stats保存在内存中。可以通过spider_stats属性访问这些stats，它是由spider domain名字作为键的一个字典。

这是Scrapy中使用的默认Stats Collector。

spider_stats

一个dicts(由spider名字作为键)，包含每个spider最后一次抓取运行的stats。

DummyStatsCollector

scrapy.statscollectors.DummyStatsCollector 这是一个类(class)

一个统计收集器什么也不做，但是非常高效(因为它什么也不做)。这个stats collector可以通过STATS_CLASS setting来设置，以禁用stats collect，以提高性能。然而，与解析页面等繁重的Scrapy工作相比，stats collection的性能损失通常微不足道。
