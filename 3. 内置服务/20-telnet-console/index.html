Telnet Console(Telnet控制台)

Scrapy附带一个内置的telnet console，用于检查和控制一个Scrapy运行过程。telnet console只是在Scrapy进程内部运行的一个常规python shell，因此您可以从它做任何事情。

telnet console是一个内置的Scrapy扩展，默认情况下是启用的，但如果需要，也可以禁用它。有关扩展本身的更多信息，请参阅Telnet console扩展。

如何访问telnet console

telnet console监听TELNETCONSOLE_PORT setting中定义的TCP端口，该设置默认为6023。要访问控制台，您需要输入:

telnet localhost 6023
>>>

您需要安装telnet程序，它默认安装在Windows中和大多数Linux发行版。

telnet console中的可用变量

telnet console就像一个运行在Scrapy进程内部的常规Python shell，因此您可以通过它做任何事情，包括导入新的模块等等。

然而，telnet console带有一些为方便而定义的默认变量:

快捷方式 描述
crawler Scrapy Crawler(scrapy.crawler.Crawler对象)
engine Crawler.engine属性
spider 活动的spider
slot engine slot
extensions 扩展管理器(Crawler.extensions属性)
stats Stats Collector(Crawler.stats属性)
settings Scrapy settings对象(Crawler.settings属性)
est print engine status报告
prefs 有关内存调试(请参阅调试内存泄漏)
p pprint.pprint函数的快捷方式。
hpy 有关内存调试(请参阅调试内存泄漏)

Telnet console使用示例

下面是一些使用telnet console可以完成的示例任务:

engine status视图

您可以使用Scrapy engine的est()方法使用telnet console快速显示其状态:

telnet localhost 6023

暂停，恢复和停止Scrapy engine

暂停:

telnet localhost 6023
>>> engine.pause()
>>>

恢复:

telnet localhost 6023
>>> engine.unpause()
>>>

停止:

telnet localhost 6023
>>> engine.stop()
Connection closed by foreign host.

Telnet Console signals(信号)

scrapy.extensions.telnet.update_telnet_vars(telnet_vars) 这是一个类(class)

在telnet console打开之前发送。您可以连接到这个信号来添加、删除或更新telnet本地名称空间中可用的变量。为了做到这一点，您需要更新handler中的telnet_vars字典。

参数:

  • telnet_vars (字典) – telnet变量的字典

Telnet settings

以下是控制telnet console行为的设置:

TELNETCONSOLE_PORT

默认: [6023, 6073]

用于telnet console的端口范围。如果设置为None或0，则使用动态分配的端口。

TELNETCONSOLE_HOST

默认: '127.0.0.1'

telnet console应该监听的接口
